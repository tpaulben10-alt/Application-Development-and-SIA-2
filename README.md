📌 Description
This repository contains a compilation of laboratory activities developed using the Laravel Framework. It demonstrates progress from basic Blade templating to advanced system integration, including CRUD operations, Form Validation, and API Consumption.

APPLICATION DEVELOPMENT
Activity 1: Emerging Technologies Exploration
Description: A blockchain is a digital record that stores data in blocks that are linked together in a chain. Once a block is added, it cannot be easily changed without everyone noticing. Many computers keep a copy of the chain, so no single person controls it and the information stays open and trustworthy.

Activity 2: UML DIAGRAM DESIGN PROJECT
Description: It will identify the system’s users, functions, and processes, then organize them visually through diagrams such as Use Case, Class, Activity, or Sequence Diagrams. The project helps improve system analysis, logical thinking, and software design skills while encouraging creativity and clear presentation of ideas.

📂 SIA 2
Activity : Project UML Final Project – Title & Design Proposal
 "Grilled Chicken Online Ordering System"

In this activity, I designed a system title, identified the problem that the system will solve, and explained the objectives, target users, and scope of the project. I also created a UML Use Case Diagram to show the interaction between the customer, admin, and delivery rider within the system. The activity helped me understand how UML diagrams are used in planning and designing software systems before actual development.

Activity 2: Soccer Player List System (Blade Templating)
Description: It demonstrates the basic concepts of Laravel such as routing, controller logic, Blade templating, and data passing. The system allows users to view a list of famous soccer players along with their details such as position, club, nationality, goals, and images. Users can click each player to view complete information on a separate detail page. The project also includes additional features such as a search bar, card-style user interface, and a simple CRUD admin panel for adding, editing, and deleting player records.

Activity 3: Chicken Breed Collection System (Full CRUD)
Description: Users can add, view, edit, search, and delete chicken breed records. The system includes database migration, controller logic, Blade templates, image upload functionality, validation, and pagination. The purpose of the activity is to demonstrate understanding of Laravel database configuration, migrations, routing, controllers, and full CRUD operations using a dynamic database-driven application.

Activity 4: Chicken Order Form System (Form Handling & Validation)
Description: The system allows users to submit chicken orders by entering details such as name, email, quantity, flavor, payment method, and address. Students will apply Laravel routes, controllers, Blade templates, and validation rules to ensure that only valid information is submitted. The activity also includes displaying validation errors, retaining old input values, and showing a success message after successful form submission.

Activity 5: Integration App (Breeze Auth & APIs)
Description: Demonstrates a simple authenticated web application with user registration/login, dashboard/profile pages, and database-backed user management.
